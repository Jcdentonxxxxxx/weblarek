Структура проекта:
- src/ — исходные файлы проекта

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Слой данных - Model

### Данные

Интерфейс `IProduct` является базовым экземпляром для учета товаров на главной странице и товаров добавленных в корзину.
```
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```
`id: string` - уникальный идентификатор товара. Обязательное поле. Тип строка.  
`description: string` - описание товара. Обязательное поле. Тип строка.  
`image: string` - название файла изображения. Обязательное поле. Тип строка.  
`title: string` - название товара. Обязательное поле. Тип строка.  
`category: string` - категория товара. Обязательное поле. Тип строка.  
`price: number | null` - цена товара. Обязательное поле. Тип строка или null.  

Интерфейс `IBuyer` является базовым экземпляром для хранения данных, которые заполняет покупатель при совершении покупки товаров.
```
interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}
```
`payment: TPayment` - способа оплаты, ограниченый литеральным типом `TPayment`. Обязательное поле.  
`email: string` - почта покупателя. Обязательное поле. Тип строка.  
`phone: string` - телефон покупателя. Обязательное поле. Тип строка.  
`address: string` - адрес покупателя. Обязательное поле. Тип строка. 

Литеральный тип `TPayment` с тремя возможными значениями `card`, `cash` и `""`
```
type TPayment = "card" | "cash" | "";
```

### Модели данных

#### Класс Products
Класс отвечает за хранение товаров, которые можно купить в приложении.  

Поля класса:  
`allProducts: IProduct[]` - поле для хранения всех товаров, после запроса на сервер.  
`chosenProduct: IProduct` - поле для хранения товара, который пользователь выбрал для детального просмотра.  

Методы класса:  
`saveProducts(products: IProduct[]): void` - метод сохраняет коллекцию товаров, которая приходит в параметрах, в поле класса `allProducts`.  
`getProducts(): IProduct[]` - метод возвращает коллекцию товаров.  
`getProduct(id: string): IProduct | undefined` - метод принимает строковый id товара и возвращает один объект с товаром, который содержит принимаемый id.  
`saveChosenProduct(product: IProduct): void` - метод сохраняет в поле `chosenProduct` один товар, который пользователь выбирает для детального просмотра.  
`getChosenProduct(): IProduct | null` - метод возвращает из поля `chosenProduct` товар.  

#### Класс Basket
Класс отвечает за хранение товаров, которые пользователь выбрал для покупки.  

Поля класса:  
`chosenProducts: IProduct[]` - поле для хранения добавленных в корзину товаров.  

Методы класса:  
`getChosenProducts():IProduct[]` - метод возвращает коллекцию товаров, добавленных в корзину.  
`addProduct(product: IProduct): void` - метод добавляет в корзину товар. Предпологается, один и тот же товар не может быть добавлен несколько раз. Для этого в методе присутствует защита.  
`removeProduct(product: IProduct): void` - метод удаляет из корзины товар. В параметрах принимает объект с товаром.  
`clearBasket(): void` - метод очищает корзину.  
`getTotalPrice(): number` - метод возвращает общую стоимость товаров, добавленных в корзину.  
`getQuantityProducts(): number` - метод возвращает количество товаров находящихся в корзине.  
`isProductInCart(id: string): boolean` - проверяет по `id: string`, есть ли в корзине данный товар.  

#### Класс Buyer
Класс отвечает за данные покупателя, которые тот должен указать при оформлении заказа.  

Поля класса:  
`payment: TPayment = ""` - поле для хранения способа оплаты, выбранного покупателем. Ограничено литеральным типом с тремя значениями `card`, `cash` и `""`.  
`address: string = ""` - поле для хранения адреса покупателя.  
`phone: string = ""` - поле для хранения телефона покупателя.  
`email: string = ""` - поле для хранения почты покупателя.  

Методы класса:  
`setProperties(properties: IBuyer): void` - метод для сохранения данных в полях класса. Предпологается, что в параметрах будет приходить объект соответствующий интерфейсу `IBuyer`, где будут присутствовать все ключи, даже если они еще не заполнены пользователем, т.е будут равны пустой строке.  
`getProperties(): IBuyer` - метод возвращает объект соответствующий интерфейсу `IBuyer` с полями класса.  
`clearProperties(): void` - метод приводит поля класса к дефолтном виду, т.е все поля будут равны пустой строке.  
`validationProperties(): object` - метод делает проверку полей, а именно является ли поле пустой строкой. Если есть пустые поля возвращает объект с описанием, какое поле не прошло валидацию. Если все поля заполнены, возвращает пустой объект.  


## Слой коммуникации

### Данные

Тип `TBodyPost` описывает данные, которые будут переданы в метод `post` класса `Connection`, при совершении покупки покупателем. Пересекается с интерфейсом `IBuyer`, переопределяет поле `payment` и добавляет `total` и `items`.  
```
export interface IBodyPost extends IBuyer {
  payment: Exclude<TPayment, "">;
  total: number;
  items: string[];
}
```
`payment: Exclude<TPayment, "">` - способ оплаты, который выбрал покупатель. Ограничен литеральным типом, у которого исключена пустая строка, так как при отправке поле обязательно должно быть заполнено.  
`total: number` - общая стоимость покупки.  
`items: string[]` - коллекция строк, в данно случае массив строк `id`


Тип `TPostResponse` описывает данные, которые будут получены, при удачном запросе метода `post` класса `Api`, при совершении покупки покупателем.  
```
export interface IPostResponse {
  id: string;
  total: number;
}
```
`id: string` - id товара в виде строки.   
`total: number` - `total: number` - общая стоимость покупки.  

Тип `TBodyGet` описывает данные, которые будут получены, при удачном запросе метода `get` класса `Api`.  
```
export interface IBodyGet {
  total: number;
  items: IProduct[];
}
```
`total: number` - общее количество товаров в ответе.  
`items: IProduct[]` - коллекция товаров в виде объектов соостветствующих интерфейсу `IProduct`.  

### Модели данных

#### Класс Connection
Класс использует композицию, чтобы выполнять запросы на сервер с помощью методов класса `Api`.  

Конструктор:  
`constructor(api: IApi)` - В конструктор передается экземпляр класса, соответствующий интерфейсу `IApi`.

Поля класса:  
`api: IApi` - в поле записывается экземпляр класса, который пердается в параметрах при создании экземпляра самого класса `Connection`.  

Методы класса:  
`get(uri: string = '/product/')` - принимает опциональный параметр(эндпоинт) в виде строки. Используется для получения списка товаров с сервера. Внутри себя вызвает метод `get` экземпляра класса, который хранится в поле `api: IApi`.  
`post(data: TBodyPost, uri: string = '/order/')` - метод для отправки на сервер данных о покупателе и выбранных им товров, в момент совершения покупки. Принимает один обязательный параметр объект соответсвующий типу `TBodyPost`. Второй параметр - эндпоинт в виде строки. Опционален. Внутри себя вызвает метод `post` экземпляра класса, который хранится в поле `api: IApi`.
